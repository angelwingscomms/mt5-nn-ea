# Agent Guide

This repo is a self-contained MT5 + Python pipeline. Use this file as the quick map.

## Architecture Snapshot

- `nn.py` is the source of truth for training, feature engineering, bar construction, and ONNX export.
- `live.mq5` consumes the ONNX model and reproduces feature scaling in MT5.
- `shared_config.mqh` is the shared config template. Symbol overrides live in `models/<SYMBOL>/config/shared_config.mqh`.
- `model_config.mqh` is auto-generated by `nn.py` and contains feature indices + normalization stats.

## Gold Profiles

Gold profiles are controlled from `nn.py`:

- `-gold` uses the legacy LSTM+MHA+GAP head and caps to the old bar count.
- `-gold-new` uses a CNN+GRU+attention pooling head.
- Both rely on fixed tick bars (`PRIMARY_TICK_DENSITY=27`) and USDX/USDJPY returns.

Data export for gold is handled by `data_gold.mq5`, which aligns USDX/USDJPY ticks by timestamp.

## Extending Profiles

To add a new profile:

1. Add a model class in `sequence_models.py`.
2. Wire a new CLI flag in `nn.py` and map it in `resolve_architecture`.
3. Add any new features in `compute_features` and update `live.mq5`.
4. Update `README.md` and this file.

## Known Constraints

- Gold profiles must be trained from a CSV that includes `usdx_bid` and `usdjpy_bid`; `nn.py` now rejects incomplete files instead of silently faking those series.
- The old `14,856` bar count came from `144` ticks/bar. Using the same bar cap at `27` ticks/bar is a shorter raw history and should be treated as a window-count match, not a full tick-span match.
- `nn.py` enforces leakage-safe train/val/test splits by embargoing `SEQ_LEN` + `TARGET_HORIZON`.
